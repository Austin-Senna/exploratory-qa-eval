# Re-import Kramabench into `other-benchmarks/`, LakeQA-shaped this time

## Context

The first import (commit `d4afbb5`) was structurally wrong. It bucketed by
`(k=#subtasks, d=difficulty)` and synthesized `node.fact` as step descriptions
("Subtask step: … | Data sources: … | Expected answer type: …"). The user
correctly flagged that I'd missed two key Kramabench folders (`solutions/`,
`dr-input/`) and that the convention is fundamentally different from what I
built.

**The right mapping, verified against actual LakeQA tasks (`tasks_mini/`):**

| LakeQA convention | Kramabench analog |
|---|---|
| `k = #reasoning hops` | `k = #subtasks` (each Kramabench subtask = one hop) |
| `d = max #nodes in any single hop` (fanout) | `d = max #data_sources in any single subtask` |
| `#nodes = sum of per-hop fanout` | `#nodes = Σ (files per subtask)` |
| `node.source = "datagov/{slug}/files/{filename}"` (or `wikipedia/{title}/content.txt`) | `node.source = "kramabench/{domain}/files/{filename}"` |
| `node.fact` = pandas/boto3 code OR short NL fact | `node.fact` = code slice from `solutions/{domain}/{task_id}.py` |
| `node.subquestion` = NL question for this hop | `subtasks[i].query` (verbatim) |
| `node.answer` = stringified intermediate | `subtasks[i].answer` (stringified) |
| `datasets_used` = unique dataset slugs | unique filenames across all subtasks |
| `reasoning_hops` = `[{hop, node_ids[], reasoning, answer}, …]` | one entry per subtask, `node_ids` = nodes generated by that subtask |

**Key correction:** `k` is **hops**, not nodes. A subtask that reads 3 files contributes 3 nodes to ONE hop, not 3 hops. Worked example for `biomedical-hard-1` (6 subtasks, fanout pattern `[2,1,1,1,1,1]`): bucket = `k-6-d-2`, total nodes = 7, hop 1 has `node_ids=[1,2]`, hops 2–6 are single-node.

## Implementation

### `convert.py` — top-level orchestration

Mirrors the structure of `data-imports/mmqa/convert.py:298-355` (reset → bucket → write loop). Per-task pipeline:

1. **Inputs:** for each domain in `("archeology", "astronomy", "biomedical", "environment", "legal", "wildfire")`:
   - Load `Kramabench/workload/{domain}.json` (list of tasks).
   - For each task `t`:
     - Load `Kramabench/solutions/{domain}/{t.id}.py` (text).
     - Build the task's canonical file set (see "Canonical file set" below).
     - Compute per-(subtask, file) slices (see "Slicer" below).
     - Build nodes, hops, reasoning chain, datasets_used, _provenance.
     - Compute `(k, d)` bucket: `k = len(t.subtasks)`, `d = max(len(unique_files(s.data_sources)) for s in t.subtasks)`.
2. **Output:** `other-benchmarks/tasks/kramabench/k-{k}-d-{d}/task_{N}.json`. Buckets wiped before writing. Tasks sorted by `id` within bucket for deterministic numbering. Same `reset_bucket()` pattern as MMQA.
3. **Summary print:** total tasks, per-bucket counts, per-task slicer outcome counters (`{ast_sliced: N, glob_loop: M, whole_solution_fallback: P}`).

### Canonical file set

For each subtask, normalize `data_sources` into a clean list of "file references":

```python
def normalize_subtask_files(subtask, parent_task) -> list[str]:
    raw = subtask.get("data_sources")
    # 1. Bare string → wrap to list
    if isinstance(raw, str):
        raw = [raw]
    # 2. None / missing → empty
    raw = raw or []
    # 3. Drop placeholders ('.', './')
    files = [f for f in raw if f and f not in (".", "./")]
    # 4. Empty after cleanup → use the parent task's top-level data_sources
    #    (this is the "summary subtask" case — biomedical-hard-1 subtask 1
    #    answers "which files contain X" with both file names but lists './'
    #    in data_sources)
    if not files:
        files = [f for f in (parent_task.get("data_sources") or [])
                 if f and f not in (".", "./")]
    return files
```

Each entry can be a literal filename, a path with subdirectories, OR a glob pattern (e.g. `"State MSA Identity Theft Data/*"`). The slicer treats globs as one virtual file → one node.

### Slicer — `slicer.py`

AST-based, stdlib only (`ast` module). The hard part of this whole import.

**Algorithm:**

1. Parse `solutions/{domain}/{task_id}.py` with `ast.parse()`. Walk the module body — each top-level statement (Import, Assign, FunctionDef, Expr, For, If, etc.) is a "chunk".

2. For each chunk, compute:
   - `vars_assigned`: targets of Assign / AugAssign / NamedExpr (collected via `ast.walk` over the LHS)
   - `vars_used`: Name nodes in Load context (everywhere except the LHS)
   - `string_literals`: every str.Constant in the chunk's subtree
   - `mentions_file(filename)`: True iff `filename` (or its basename) is a substring of any string_literal in this chunk

3. For each unique filename `F` in the task's union-of-subtask-files:
   - **Root chunks:** chunks where `mentions_file(F)` is True AND a `pd.read_*` / `pd.read_excel` / `open(...)` / `gpd.read_file(...)` / `pd.read_html(...)` / `BeautifulSoup(...)` call appears in the chunk (heuristic: chunk's `vars_used` ∪ Call-target-names contains a known reader).
   - **Tracked vars for F:** start = root chunks' `vars_assigned`. Walk forward through subsequent chunks; if a chunk's `vars_used` ⊆ tracked vars for F (and not in any other file's tracked set), append it to F's slice and add its `vars_assigned` to F's tracked set.
   - **Stop condition:** stop walking forward as soon as a chunk's `vars_used` overlaps with another file's tracked vars (this is integration code, see step 4).
   - **F's slice:** the ordered list of chunks. Render via `ast.unparse(...)` (Python 3.9+) joined with newlines, prepend `import` statements that the slice references, prepend `data_path = "..."` if used.

4. **Integration tail:** chunks whose `vars_used` references variables tracked by ≥2 files. These get attached to the terminal node (last node of last subtask) by appending after that node's slice, with a comment marker `# --- integration ---`.

5. **Glob handling:** if a filename ends with `*` or contains `?` / `[`, look for chunks that contain a `glob.glob(...)` / `os.listdir(...)` / `Path(...).glob(...)` / `pd.concat([...for...in...])` matching the directory part. The matched chunks form the slice. Mark `_provenance.fact_origin = "glob_loop"` for nodes whose source is a glob pattern.

6. **Fallback paths** (counted in summary):
   - **No `pd.read_*` references the file** (file mentioned only in a comment, or referenced via a variable computed elsewhere): use the **whole solution** as the fact. Mark `_provenance.fact_origin = "whole_solution_fallback"`.
   - **Solution file doesn't exist** (e.g. workload claims a task but no `solutions/{domain}/{task_id}.py`): set fact = `f"# Solution file missing for {task_id}; see workload entry for step description: {step}"`. Mark `_provenance.fact_origin = "missing_solution"`.
   - **Slicer crashed on a SyntaxError** (defensive — Kramabench solutions have all parsed in spot-checks but be safe): same as missing-solution fallback. Mark `_provenance.fact_origin = "syntax_error"`.

**Slicer output API (signatures):**

```python
def slice_solution(solution_text: str, file_set: list[str]) -> dict[str, str]:
    """Returns {filename: code_slice} for each file in file_set.
    Returns the whole solution for files that fall through to fallback."""

def integration_tail(solution_text: str, file_set: list[str]) -> str:
    """Returns the integration code (cross-file ops + final computation),
    rendered as a string. Empty if no cross-file integration was detected."""
```

**Limitations I'm not solving in this round:**
- Solutions that compute filenames dynamically (e.g. `for state in states: pd.read_csv(f"./{state}.csv")`) — handled via glob-pattern detection if the workload uses a glob; otherwise falls into `whole_solution_fallback`.
- Solutions with helper functions that obscure the read site (e.g. `read_clean_numeric_csv("./Ohio.csv")`) — the heuristic recognizes the call as a "reader" if the function name matches the regex `^(read|load|fetch|parse)_`. Other helper names → fallback.
- Cross-file variable references inside a slice (e.g. `mmc2.xlsx` slice references `case_ids` from `mmc1.xlsx`'s slice) — left as-is; the slice is honest about its dependencies via a comment header `# Depends on prior nodes: case_ids` (computed by diffing slice's `vars_used` against `vars_assigned`).

### Per-task assembly

For each task `t`:

```python
nodes = {}                # {"1": {source, fact, subquestion, answer}, ...}
reasoning_hops = []       # [{hop, node_ids, reasoning, answer}, ...]
reasoning_chain = []      # ["Node 1: ...", ..., "Final answer: ..."]
datasets_used = []        # ordered unique filenames

next_node_id = 1
for hop_idx, subtask in enumerate(t["subtasks"], start=1):
    files = normalize_subtask_files(subtask, t)
    if not files:
        # Synthesize a single "no-file" node (rare; only happens when even
        # parent's data_sources is empty). Source = kramabench/{domain}/files/step-{hop_idx}
        ...
    hop_node_ids = []
    for fname in files:
        slice_text = slicer_output[fname]  # may be the whole solution if fallback
        nodes[str(next_node_id)] = {
            "source": f"kramabench/{domain}/files/{fname}",
            "fact": slice_text,
            "subquestion": subtask["query"],
            "answer": stringify(subtask["answer"]),
        }
        reasoning_chain.append(f"Node {next_node_id}: subtask {hop_idx} on {fname}")
        if fname not in datasets_used:
            datasets_used.append(fname)
        hop_node_ids.append(next_node_id)
        next_node_id += 1
    reasoning_hops.append({
        "hop": hop_idx,
        "node_ids": hop_node_ids,
        "reasoning": f"Subtask {hop_idx}: {subtask.get('step', subtask['query'])}",
        "answer": stringify(subtask["answer"]),
    })

# Append integration tail to terminal node's fact (terminal node = last node of last hop)
if integration_tail_text:
    last_id = str(next_node_id - 1)
    nodes[last_id]["fact"] = nodes[last_id]["fact"] + "\n\n# --- integration ---\n" + integration_tail_text

# Override terminal node's answer with task-level answer if subtask diverges
nodes[str(next_node_id - 1)]["answer"] = stringify(t["answer"])
reasoning_chain.append(f"Final answer: {stringify(t['answer'])}")
```

`_provenance` per task:
```json
{
  "benchmark": "kramabench",
  "split": "main",
  "source_id": "<t.id>",
  "domain": "<domain>",
  "difficulty": "<easy|medium|hard from id parse>",
  "runtime": <t.runtime>,
  "answer_type": "<t.answer_type>",
  "solution_path": "Kramabench/solutions/<domain>/<t.id>.py",
  "raw_data_root": "Kramabench/data/<domain>/input",
  "fact_origin_per_node": {"1": "ast_sliced", "2": "ast_sliced", ...}
}
```
